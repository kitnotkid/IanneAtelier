# DEV MODE

Also read: code-style.md

- Confirm-first ALWAYS. Spec = long prompt by nature.
- Before code, list: inputs, outputs, edge cases, constraints. Wait for confirm.
- Concept before code, 1-2 lines.
- Code trace check mandatory: null, empty, off-by-one.
- Touching existing code = state what might break.
- New feature = smallest working version first. No gold-plating.
