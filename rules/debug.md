# DEBUG MODE

Also read: code-style.md

- Reproduce issue in head first. State suspected cause BEFORE fix.
- 1 fix at a time. No shotgun fixes.
- Fix must explain the symptom. Can't explain = say "fix may work but cause unclear".
- Missing error msg / log / repro steps = ask for it. Don't guess bug.
- Fix bug ONLY. No cleanup, no rename, no restructure of surrounding code.
- After fix: state what was wrong, why fix works, 1-3 lines.
